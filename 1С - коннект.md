# <q>Добавить коннектор к 1С через API или выгрузку CSV → агент подтягивает обороты и делает сверку</q> Расскажи мне подробно

Отлично, у меня достаточно данных для подробного разбора. Погнали.

***

Есть три варианта коннектора к 1С — от простого к сложному. Для задачи «агент подтягивает обороты и делает сверку» оптимальна связка **OData API + Python-скрипт + Claude**. Ниже — полная архитектура с конкретными кодами.

## Три способа подключить 1С

| Способ | Трудоёмкость | Когда применять |
| :-- | :-- | :-- |
| **CSV-выгрузка вручную** | Минимум, нет кода | У клиента нет IT-доступа к серверу 1С, разовые задачи |
| **OData REST API** (1С 8.3+) | Средняя, 1-2 дня | Облачная 1С (1cfresh) или собственный сервер с веб-интерфейсом |
| **HTTP-сервис 1С (кастомный)** | Высокая, нужен 1С-разработчик | Нестандартные запросы, сложные отчёты, on-premise |

Большинству клиентов МСБ подходит **OData** — он включён в 1С:Бухгалтерию 8.3+ по умолчанию.[^1]

## Путь 1: CSV-выгрузка (быстрый старт)

Клиент делает выгрузку сам: открывает оборотно-сальдовую ведомость → `Действия → Экспорт списка → Текст CSV (разделители — точка с запятой)`.[^2]

Ты получаешь файл и запускаешь агента через Claude API с прикреплённым файлом. Это уже работает сейчас, без настройки серверов.

**Что делает агент с CSV:**

```python
import anthropic, pandas as pd

def analyze_osv(csv_path: str) -> str:
    df = pd.read_csv(csv_path, sep=';', encoding='cp1251')
    
    client = anthropic.Anthropic()
    message = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=4096,
        messages=[{
            "role": "user",
            "content": f"""Ты CFO-ассистент. Проанализируй оборотно-сальдовую ведомость.
            
Данные ОСВ:
{df.to_string()}

Выполни:
1. Проверь баланс: сумма дебетовых оборотов = сумма кредитовых
2. Выяви счета с нулевыми или отрицательными остатками (аномалии)
3. Сравни обороты по 51 и 62 счетам — есть ли кассовые разрывы
4. Проверь остатки по 60/76 — есть ли старая кредиторка (>90 дней)
5. Сформируй список вопросов к клиенту на основе аномалий"""
        }]
    )
    return message.content[^0].text
```

## Путь 2: OData API (основной вариант)

### Как включить в 1С

В 1С:Предприятие 8.3: `Администрирование → Публикация на веб-сервере → Включить OData REST API`. После этого интерфейс доступен по адресу:[^3][^4]

```
http://<сервер>/имя_базы/odata/standard.odata/
```

Для 1cfresh.com (облачная 1С):

```
https://msk1.1cfresh.com/a/ea/<ID_БАЗЫ>/odata/standard.odata/
```


### Python-клиент для получения оборотов

```python
import requests
import pandas as pd
from datetime import date

class OneCConnector:
    def __init__(self, base_url: str, login: str, password: str):
        self.base_url = base_url.rstrip('/')
        self.auth = (login, password)
        self.headers = {'Accept': 'application/json;odata=verbose'}
    
    def get(self, endpoint: str, params: dict = None) -> dict:
        url = f"{self.base_url}/odata/standard.odata/{endpoint}"
        resp = requests.get(url, auth=self.auth, 
                           headers=self.headers, params=params)
        resp.raise_for_status()
        return resp.json()
    
    def get_counterparties(self) -> pd.DataFrame:
        """Получить контрагентов"""
        data = self.get("Catalog_Контрагенты", {
            '$select': 'Ref_Key,Description,ИНН,КПП',
            '$filter': 'DeletionMark eq false',
            '$format': 'json'
        })
        return pd.DataFrame(data['value'])
    
    def get_account_turnovers(
        self, 
        account: str, 
        date_from: str, 
        date_to: str
    ) -> pd.DataFrame:
        """Получить обороты по счёту из регистра бухгалтерии"""
        # AccountingRegister_Хозрасчетный — стандартное имя в БП 3.0
        data = self.get("AccountingRegister_Хозрасчетный_RecordType", {
            '$filter': (
                f"AccountDr_Key eq guid'{account}' and "
                f"Period ge datetime'{date_from}T00:00:00' and "
                f"Period le datetime'{date_to}T23:59:59'"
            ),
            '$select': 'Period,AccountDr,AccountCr,Amount,Content',
            '$format': 'json'
        })
        return pd.DataFrame(data.get('value', []))
```


### Ключевые объекты OData в 1С:Бухгалтерия 3.0

| Что нужно | OData endpoint |
| :-- | :-- |
| Контрагенты | `Catalog_Контрагенты` |
| Движения по счетам | `AccountingRegister_Хозрасчетный_RecordType` |
| Документы «Поступление» | `Document_ПоступлениеТоваровУслуг` |
| Документы «Реализация» | `Document_РеализацияТоваровУслуг` |
| Банковские выписки | `Document_СписаниеСРасчетногоСчета` |
| Акты сверки | `Document_АктСверкиВзаиморасчетов` |

## Архитектура агента сверки

Вот полная схема, как это работает в связке с Claude:

```python
import anthropic
from typing import Optional

def run_reconciliation_agent(
    connector: OneCConnector,
    period_from: str,
    period_to: str,
    counterparty_inn: Optional[str] = None
):
    """
    Агент сверки: подтягивает данные из 1С и делает анализ через Claude
    """
    
    # 1. Забираем данные из 1С
    turnovers_60 = connector.get_account_turnovers('60', period_from, period_to)
    turnovers_62 = connector.get_account_turnovers('62', period_from, period_to)
    counterparties = connector.get_counterparties()
    
    # 2. Готовим контекст для агента
    context = f"""
    Период: {period_from} — {period_to}
    
    Обороты по счёту 60 (поставщики):
    {turnovers_60.to_markdown(index=False)}
    
    Обороты по счёту 62 (покупатели):
    {turnovers_62.to_markdown(index=False)}
    
    Справочник контрагентов:
    {counterparties[['Description','ИНН']].to_markdown(index=False)}
    """
    
    # 3. Запускаем Claude с инструментами
    client = anthropic.Anthropic()
    
    tools = [{
        "name": "flag_anomaly",
        "description": "Зафиксировать аномалию для включения в отчёт",
        "input_schema": {
            "type": "object",
            "properties": {
                "account": {"type": "string", "description": "Номер счёта"},
                "counterparty": {"type": "string"},
                "amount": {"type": "number"},
                "issue_type": {
                    "type": "string",
                    "enum": ["overpayment", "overdue_debt", 
                             "missing_document", "balance_mismatch"]
                },
                "description": {"type": "string"},
                "priority": {"type": "string", "enum": ["high", "medium", "low"]}
            },
            "required": ["account", "issue_type", "description", "priority"]
        }
    }]
    
    messages = [{
        "role": "user",
        "content": f"""Ты CFO-ассистент. Выполни сверку взаиморасчётов.

{context}

Задачи:
1. Найди контрагентов с дебиторкой старше 60 дней (сч.62)
2. Найди контрагентов с кредиторкой без закрывающих документов (сч.60)
3. Проверь, нет ли задвоений платежей
4. Для каждой аномалии вызови инструмент flag_anomaly
5. Сформируй итоговое заключение: что срочно, что на контроль"""
    }]
    
    # Агентный цикл с инструментами
    anomalies = []
    while True:
        response = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=4096,
            tools=tools,
            messages=messages
        )
        
        if response.stop_reason == "end_turn":
            break
        
        # Обработка вызовов инструментов
        tool_results = []
        for block in response.content:
            if block.type == "tool_use" and block.name == "flag_anomaly":
                anomalies.append(block.input)
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": "Аномалия зафиксирована"
                })
        
        if tool_results:
            messages.append({"role": "assistant", "content": response.content})
            messages.append({"role": "user", "content": tool_results})
        else:
            break
    
    # Итоговый текст от агента
    conclusion = next(
        (b.text for b in response.content if hasattr(b, 'text')), ""
    )
    
    return {"anomalies": anomalies, "conclusion": conclusion}
```

## Интеграция через n8n (без кода)

Если не хочешь писать Python — есть готовый путь через n8n:[^5]

1. **HTTP Request node** → OData 1С (GET, Basic Auth)
2. **Code node** → парсишь JSON, формируешь таблицу
3. **Anthropic node** → передаёшь данные в Claude с промптом сверки
4. **Google Docs / Telegram node** → отправляешь отчёт клиенту или себе

Есть и готовый no-code коннектор через ApiMonster: настраиваешь вебхук в 1С → ApiMonster → Claude без единой строчки кода.[^5]

## Практический план запуска

1. **День 1** — проверь, включён ли OData: открой в браузере `http://адрес_1С/база/odata/standard.odata/?$format=json`. Если вернул JSON с метаданными — всё работает
2. **День 2** — запусти Python-клиент, вытяни список контрагентов. Убедись, что аутентификация проходит
3. **День 3** — напиши первый запрос оборотов по счёту 60/62 за последний месяц
4. **День 4-5** — подключи Claude, напиши промпт под конкретного клиента, протестируй на реальных данных
5. **День 6** — оформи как скрипт с параметрами (клиент, период), добавь генерацию PDF-отчёта

Если у клиента 1С установлена локально (не облако) — нужно либо поднять веб-сервер на IIS/Apache, либо использовать COM-объект через Windows-машину, либо договориться о регулярной CSV-выгрузке через Dropbox/Яндекс.Диск.[^6]
