# ADR-0002: Bank = источник правды по остатку; 1С = источник правды по обязательствам

**Статус:** Принято  
**Дата:** 2026-05-23  
**Авторы:** Engineering Review

---

## Контекст

У клиента два источника финансовых данных:
- **Банковская выписка:** факт движения денег, остаток на счёте.
- **1С:** управленческие регистры, ДДС, дебиторка, кредиторка, статьи расходов.

Нередко цифры расходятся (timing, проводки не синхронизированы). Нужно решить, какому источнику доверять и когда.

---

## Решение

| Вопрос | Источник правды |
|---|---|
| Остаток на расчётном счёте | Банк (выписка) |
| Факт движения денег | Банк (выписка) |
| Плановые обязательства / кредиторка | 1С |
| Дебиторка (ожидаемые поступления) | 1С |
| Статьи ДДС (категоризация) | 1С |

---

## Обоснование

- Банк — единственный авторитетный источник фактического остатка (51 счёт в 1С нередко отстаёт).
- 1С — единственный источник структурированной дебиторки и обязательств с контрагентами.
- При расхождении: **reconciliation alert** (`bank_only` | `erp_only` | `amount_mismatch`), не silent overwrite.

---

## Последствия

- Sprint 1: только банк → прогноз без дебиторки, только ручные обязательства.
- Sprint 2: после подключения 1С появляется `Reconciler` с явными статусами.
- Расхождение остатка банка ≠ 51 счёт 1С на дату → alert в дашборде, не ошибка.
- Не подходит для клиентов, где 1С — единственный источник (нет выписки) → за рамками MVP.
