# Финансовый автопилот

SaaS AI CFO для МСБ в РФ: read-only интеграция с 1С и банком, ежедневный контур «деньги → сигнал → объяснение → действие». Не заменяет бухгалтерский учёт.

## Language

### Продукт и роли

**Автопилот**:
Продукт «Финансовый автопилот» — управленческий слой над 1С и банком, не регламентированный учёт.
_Avoid_: copilot (как бренд), BI-дашборд, ERP

**AI CFO**:
Класс продукта: ежедневные ответы «сколько денег», «что будет через 7–90 дней», «какие риски сегодня».
_Avoid_: chatbot, финансовый ассистент (слишком общо)

**Собственник**:
Главный потребитель ежедневной сводки: остатки, обязательства, кассовые риски.
_Avoid_: CEO (в русскоязычном контексте продукта)

**Финансовый менеджер**:
Пользователь прогноза, сценариев и анализа отклонений.
_Avoid_: CFO (как должность в UI), бухгалтер

### Управленческий учёт

**ДДС**:
Движение денежных средств — операционный, инвестиционный, финансовый контур; не путать с ОПиУ.
_Avoid_: cash flow (в пользовательском UI), денежный поток (как синоним без контекста)

**Платёжный календарь**:
Планируемые поступления и выплаты по датам; живёт от факта 1С + банка + ручных правок.
_Avoid_: график платежей, payment schedule

**Кассовый разрыв**:
Ситуация, когда запланированные выплаты превышают доступные деньги в горизонте прогноза.
_Avoid_: дефicit, нехватка ликвидности (канцелярит)

**Дебиторка**:
Дебиторская задолженность покупателей; контроль просрочки и влияния на ДДС.
_Avoid_: receivables, задолженность клиентов (без уточнения дебитор/кредитор)

**Кредиторка**:
Кредиторская задолженность поставщикам и обязательствам.
_Avoid_: payables, обязательства (слишком широко)

**Остаток**:
Доступные деньги на расчётных счетах и в кассе на дату; второй источник правды — банковская выписка.
_Avoid_: balance (в UI), сальдо (бухгалтерский термин без контекста)

### Источники данных

**1С**:
Основной источник факта: обороты, ОСВ, управленческие регистры. Read-only в MVP.
_Avoid_: ERP, учётная система (без имени)

**OData**:
Предпочтительный API-коннектор к 1С 8.3+ для автоматической выгрузки.
_Avoid_: REST API (без уточнения OData vs HTTP-сервис)

**Банковская выписка**:
Второй источник правды для сверки остатков и фактических движений.
_Avoid_: банк-клиент (как технология), statement

**ОСВ**:
Оборотно-сальдовая ведомость — базовый отчёт для сверки и CSV-импорта на старте.
_Avoid_: trial balance, оборотка (в документации для клиента)

**CSV-выгрузка**:
Ручной fallback-коннектор, когда OData недоступен; разделитель `;`, кодировка cp1251.
_Avoid_: Excel (как формат обмена), файл выгрузки (без типа)

### MVP-модули

**Деньги сегодня**:
Экран/блок: остатки, ближайшие платежи, красные флаги на сегодня.
_Avoid_: dashboard (как термин в продукте), главная

**Кассовый прогноз**:
Горизонт 7–90 дней: ожидаемые поступления, выплаты, риск разрыва.
_Avoid_: forecast (в UI), прогноз выручки

**Сигнал**:
Автоматически найденное отклонение или риск, требующий внимания.
_Avoid_: alert, уведомление (слишком технично)

**Объяснение**:
Текстовая расшифровка сигнала на естественном языке с опорой на факт.
_Avoid_: комментарий, insight

## Example dialogue

> **Собственник:** Почему на следующей неделе кассовый разрыв?
> **Автопилот:** По платёжному календарю 12.06 выходит 4,2 млн кредиторки при ожидаемом поступлении 1,8 млн. Дебиторка по счёту 62 даёт только 0,9 млн в горизонте 7 дней — основной риск в просрочке контрагента X.
