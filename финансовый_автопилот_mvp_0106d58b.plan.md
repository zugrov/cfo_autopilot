# Design Doc: Финансовый автопилот MVP

## Цель
Дать собственнику и финансисту МСБ единый экран, на котором утром за 3 минуты видно: сколько денег доступно, когда и почему будет дефицит, какие обязательства горят. Первый успех — живой клиент смотрит на свои данные в системе.

## Пользователь / Сценарий
- **Роль:** собственник + финансист (принимают решение совместно, покупают вместе)
- **Боль:** кассовый разрыв узнают в день платежа; Excel устаревает за сутки
- **Сценарий v0:** загрузить CSV банковской выписки → увидеть остатки и прогноз остатка на 30 дней на одном экране

## Scope MVP (всё из ТЗ — в планах, но приоритизировано по спринтам)

### Спринт 1 (≤2 нед.) — «Деньги и прогноз»
- Загрузка CSV/Excel банковской выписки (парсинг, нормализация)
- Главный экран: остатки по счетам, поступления/списания за 7/30 дней
- Прогноз остатка на 30 дней (экстраполяция паттернов + ручной ввод обязательств)
- Сигнал дефицита: подсветка дней с отрицательным прогнозным остатком
- Telegram-дайджест: ежедневное сообщение с остатком и ближайшими рисками

### Спринт 2 (нед. 3-4) — «Обязательства, дебиторка и объяснения»
- Ручная загрузка плана платежей (CSV/Excel)
- Aging дебиторки
- Алерты: просрочка, приближение критичного платежа, аномальный платёж
- Прогноз на 90 дней с базовым и стрессовым сценарием
- **Rule-based объяснение изменения остатка:** топ-3 причины за неделю (крупный платёж, задержка поступления, сезонный спад) — без LLM, на логике по транзакциям

### Спринт 3 (нед. 5-6) — «1С и AI-ассистент»
- Загрузка выгрузки из 1С (файлы обмена / XML / CSV от интегратора)
- AI-чат: 20 типовых финансовых вопросов по данным клиента
- Объяснимость: каждый ответ со ссылкой на источник и уровнем уверенности

### Спринт 4 (нед. 7-8) — «Полировка и onboarding»
- Ролевая модель: собственник / финансист / бухгалтер / наблюдатель
- Управленческий дайджест (PDF/email еженедельно)
- Журнал действий, read-only доступ по умолчанию

## Out of Scope (MVP)
- Мобильное приложение (только веб)
- Мультиюрлица
- Прямые банковские API-коннекторы
- Автопроведение документов в 1С
- Налоговый/зарплатный календарь
- Кредитный скоринг контрагентов

## Архитектура (high-level)

```
[Источники данных]            [Backend]              [Frontend]
CSV банк / 1С выгрузка ──►  FastAPI (Python)  ──►  Next.js / React
Ручной ввод бюджета     ──►  PostgreSQL              Один SPA
                             ETL / нормализация
                             Forecast engine
                             LLM adapter (YandexGPT / OpenAI)
                                    │
                             Telegram Bot (aiogram 3.x)
```

- **Backend:** Python + FastAPI — один разработчик, быстрый старт
- **БД:** PostgreSQL — сущности из ТЗ (юрлицо, счёт, статья ДДС, платёж, прогноз, роль)
- **Прогноз:** детерминированная модель (факт + план) + простая авторегрессия; без ML-инженера
- **AI-чат:** YandexGPT (данные остаются в РФ) с fallback на OpenAI; RAG по финансовым данным клиента
- **Telegram:** aiogram 3.x, webhook
- **Хостинг:** Yandex Cloud или Selectel (российский контур, требование ТЗ)
- **Auth:** JWT + ролевая модель, все запросы к источникам — read-only

## Ключевые сущности БД
- `company` — юрлицо
- `bank_account` — счёт
- `transaction` — факт (поступление/списание)
- `obligation` — плановое обязательство
- `receivable` — дебиторская задолженность
- `budget_line` — строка бюджета/ДДС
- `forecast_scenario` — сценарий прогноза
- `user` + `role` — ролевая модель

## Риски

- **Качество данных клиента:** CSV банков имеют разные форматы → нужен гибкий парсер с маппингом полей при загрузке
- **Доверие к AI-ответам:** клиент будет сомневаться → каждый ответ с цифрой + источником + предупреждением о неполноте данных
- **Sales cycle:** покупают через доверие → запускать с пилотом «загрузи свою выписку, посмотри на себя», не через демо на чужих данных
- **LLM-стоимость:** при ограниченном бюджете кешировать типовые ответы, использовать YandexGPT для снижения затрат
- **1С-интеграция:** у каждого клиента своя конфигурация → стартуем с файлами, не с прямым API

## Монетизация (обновлено после CEO review)

- **Базовый тир (SaaS):** платформа + автопилот без участия человека — ежемесячная подписка за компанию
- **Premium тир (CFO-light):** платформа + 2-4 часа в месяц с закреплённым финансовым куратором (читает дайджесты, даёт рекомендации) — продаётся в 5-10x дороже подписки, upsell после 30 дней использования
- Единоразовое внедрение (onboarding-fee)
- Доплата за каждое дополнительное юрлицо (Post-MVP)

## Критерии готовности MVP
- Клиент загружает CSV банковской выписки без инструкции разработчика
- На главном экране: остатки + ближайшие обязательства + сигнал дефицита
- Прогноз на 30 дней формируется автоматически, допускает ручную корректировку
- Telegram-дайджест приходит ежедневно
- AI-ассистент отвечает на 20 типовых вопросов по загруженным данным
- Из любой цифры можно провалиться в исходные транзакции
- Роли и read-only настроены по умолчанию
